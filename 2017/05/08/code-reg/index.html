发表于 2017-05-08 | 分类于 总结 | 阅读次数

2017 前端项目开发架构

项目架构

  • HTML5 + JS(ES6+) + SASS
  • JS框架 Angularjs@^1.6.4
  • HTML/CSS框架 Bootstrap@^3.x + admin-lte@^2.x + 移动优先 + 响应式开发(基于REM+Media Queries)
  • 测试框架 自动化单元测试: Karma@^1.6.0 + Jasmine@^2.6.1 自动化端到端测试: Protractor 代码覆盖率istanbul
  • 构建打包压缩混淆工具 Webpack@2.x
  • 图标字体 Font-Awesome
  • 包管理工具 Yarn
  • 项目管理工具 禅到

一 使用指南

1. 开发环境

  • VSCode (开发工具)
  • Git (版本控制)
  • Nodejs@^7.9.0 + NPM@^4.2.0 + Yarn@^0.23.3 (开发环境)
  • Webpack@^2 (打包构建工具)
  • Meld (开源好用的代码合并工具)
  • OpenVPN (Gitlab 服务器)
  • 内网VPN (用于前后端接口调试)

2. 开发部署

  • 克隆项目到本地

  • 安装所需插件

  • 启动本地项目

  • 启动本地项目, 并自动打开浏览器

  • 打包生产代码

打包会生成一个stats.json文件, 这是一个wenpack-analyse文件, 打开webpack-analyse 网站, 并上传该文件, 可以进行所使用插件的分析

  • 测试

  • 生成生产代码, 并在本地测试, 需要全局安装 http-server

3. 代码同步上传

上传代码前, 尽量在群里说一下, 避免不必要的多人同时上传代码冲突

  1. 首先配置好openvpn
  2. 以管理员权限开启openvpn(window系统, 右键以管理员权限运行)

给项目打标签(Tag)

  2. 查看tag

5. 格式化代码规范

  • 使用 vscode Beautify 插件进行代码格式化, 配置文件 .jsbeautifyrc, 更多信息
  • 缩进都控制 2字节
  • 文件内容末尾需要添加一个空行
  • 最大只允许3个空行
  • 尽量时常格式化自己写的代码, 上传之前要格式化为统一格式

三 特别注意

  • 函数一定要写注释, 各个文件要多写注释
  • 代码务必清晰易懂
  • 命名简洁直观易懂, 尽量不要使用中文拼音
  • 严格按着ESLint, scss-lint, htmlhint规范进行开发
  • 要时常merge代码,并上传
  • 主题,风格,要统一
  • 相关业务的接口,前后端命名要统一

四 OpenVPN使用说明 2017-05-03

  1. 下载附件openvpn安装包，安装openvpn，拷贝config目录到安装目录
  2. 配置Openvpn启动程序，设置兼容性为通过管理员运行(windowOS, 右单击-以管理员身份运行)
  3. 连接openvpn
  4. 访问Gitlab服务
  5. 配置ssh-key
  6. 相关配置完成后，需要下载本地代码库

五 自动化单元测试(Auto Unit Testing)

  1. 全局安装karma[可选]
